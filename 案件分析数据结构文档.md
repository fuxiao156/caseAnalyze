## 数据结构总览

```typescript
interface CaseAnalysisData {
  title: string;                    // 案件标题
  detail: string;                    // 案件详细描述
  事件画像: EventTag[];              // 事件标签画像
  核心成因分析: CauseAnalysis[];     // 核心成因及占比
  人物维度数据: PersonDimension;     // 人物维度分析
  时间维度数据: TimeDimension;       // 时间维度分析
  信息维度数据: InformationDimension; // 信息维度分析
  驱动力维度数据: DriverDimension;   // 驱动力维度分析
}
```

## 字段详细说明

### 1. title (案件标题)

**类型**: `string`  
**必填**: 是  
**说明**: 案件的完整标题，应简洁明了地概括案件的核心内容  
**示例**: `"岳阳市平江县刘某与某矿石选厂劳动争议纠纷调解案"`  

---

### 2. detail (案件详细描述)

**类型**: `string`  
**必填**: 是  
**说明**: 案件的完整描述文本，包含案件背景、调解过程、调解结果和案例点评等完整信息  

---

### 3. 事件画像 (EventTag[])

**类型**: `Array<EventTag>`  
**必填**: 是  
**说明**: 用于快速标识案件类型的标签集合，用于事件分类和快速识别

**EventTag 结构**:
```typescript
interface EventTag {
  tag: string;  // 标签文本
}
```

**要求**:
- 标签数量：1-4 个

**示例**:
```json
[
  { "tag": "工伤争议" },
  { "tag": "劳动纠纷" },
  { "tag": "社保投缴" },
  { "tag": "历史遗留" }
]
```

**说明**: 分析导致案件发生或难以解决的核心原因，并给出每个原因的权重占比

**CauseAnalysis 结构**:
```typescript
interface CauseAnalysis {

}
```

**要求**:
- 成因数量：2-4 个
- 所有成因的 `percentage` 总和应接近 100

**示例**:
```json

  { "tag": "伤残待遇未兑现", "percentage": 35.0 },
  { "tag": "政策执行难度大", "percentage": 25.0 },
```

### 6. 时间维度数据 (TimeDimension)

**类型**: `TimeDimension`  
**必填**: 是  
**说明**: 按时间线梳理案件的关键节点，分析每个节点的影响因素

**TimeDimension 结构**:
```typescript
interface TimeDimension {
  summary: string;      // 维度总结说明
  timeline: TimeNode[]; // 时间节点列表
}

interface TimeNode {
  date: string;              // 日期（格式：YYYY-MM 或 YYYY-MM-DD）
  event: string;             // 事件描述
  nodeFactors: NodeFactor[]; // 该节点的关键要素分析
}

interface NodeFactor {
  name: string;  // 要素名称（如"经济要素"、"法律要素"、"心理要素"、"社会要素"、"政策要素"等）
  content: string; // 要素内容描述
}
```

**要求**:
- `summary`: 50-100 字，概括时间维度的分析价值
- `timeline`: 3-8 个关键时间节点，按时间顺序排列
- `date`: 使用标准日期格式，精确到月份或日期
- `event`: 简洁描述该时间节点发生的关键事件（10-20 字）
- `nodeFactors`: 每个节点包含 1-4 个要素分析
- 常见要素类型：
  - **经济要素**: 涉及资金、补偿、费用等经济相关的影响
  - **法律要素**: 涉及法律依据、协议签订、法律程序等
  - **心理要素**: 涉及当事人情绪、心理状态、认知变化等
  - **社会要素**: 涉及社会关系、外部介入、社会影响等
  - **政策要素**: 涉及政策变化、政策利用、政策限制等
- `content`: 每个要素描述 50-150 字，应具体说明该要素在该时间节点如何影响案件发展

```json

          "name": "心理要素",
          "content": "刘某被确诊后心理极度恐惧，对未来生活感到绝望，情绪波动剧烈。"
        }
      ]
    }
  ]
}
```

### 7. 信息维度数据 (InformationDimension)

**类型**: `InformationDimension`  
**必填**: 是  
**说明**: 通过对比当事人的主观认知与客观事实，识别导致调解僵局的关键信息断层和认知偏差

**InformationDimension 结构**:
```typescript
interface InformationDimension {
  summary: string;  // 维度总结说明
  items: InfoItem[]; // 信息对比项列表
}

interface InfoItem {
  category: string;      // 认知类别（如"社保性质认知"、"补偿范围认知"、"解决路径认知"等）
  subjective: SubjectiveInfo; // 主观认知
  objective: ObjectiveInfo;     // 客观事实
}

interface SubjectiveInfo {
  text: string;   // 当事人的主观认知描述（通常为第一人称表述）
  tags: string[]; // 主观认知的特征标签（如"受害者心态"、"故意猜测"、"对抗思维"等）
}

interface ObjectiveInfo {
  text: string;   // 客观事实描述
  tags: string[]; // 客观事实的特征标签（如"政策瓶颈"、"物理受阻"、"法律空白"、"合理主张"等）
}
```

**要求**:
- `summary`: 50-100 字，说明信息维度分析的核心价值
- `items`: 3-6 个信息对比项，每个对比项聚焦一个关键的认知偏差点
- `category`: 明确标识认知的类别，常见类别包括：
  - 对权利/义务的认知
  - 对政策/法律的认知
  - 对解决路径的认知
  - 对对方动机的认知
  - 对自身处境的认知
- `subjective.text`: 使用第一人称或直接引语形式，体现当事人的主观感受和判断（50-100 字）
- `subjective.tags`: 1-3 个标签，概括主观认知的心理特征
- `objective.text`: 客观、中性的描述，说明实际情况（50-100 字）
- `objective.tags`: 1-3 个标签，概括客观事实的特征
- 主观与客观应形成鲜明对比，突出信息不对称导致的误解

**示例**:
```json
{

    {
      "category": "社保性质认知",

      "objective": {

      }
    }
  ]
}
```

### 8. 驱动力维度数据 (DriverDimension)

**类型**: `DriverDimension`  
**必填**: 是  
**说明**: 通过分析驱动力与约束力的博弈，模拟案件从诱发到化解的动态平衡过程

**DriverDimension 结构**:
```typescript
interface DriverDimension {
  summary: string;  // 维度总结说明
  states: DriverState[]; // 驱动力状态列表
}

interface DriverState {
  name: string;           // 状态名称（如"失衡诱发"、"动力干预"、"归因闭环"）
  status: string;         // 状态描述
  leftWeights: Weight[];  // 左侧权重（驱动力）
  rightWeights: Weight[]; // 右侧权重（约束力）
  analysis?: string[];    // 分析说明（可选，通常只在"归因闭环"状态中包含）
}

interface Weight {
  name: string;      // 权重项名称
  value: number;      // 权重值（数值，用于可视化展示）
  describe: string;  // 权重项描述
}
```

**要求**:
- `summary`: 50-100 字，说明驱动力维度分析的核心价值
- `states`: 通常包含 3 个状态，按时间顺序：
  1. **失衡诱发**: 案件发生前的状态，驱动力显著超过约束力
  2. **调节介入**: 调解介入后的状态，引入外部干预变量
  3. **归因闭环**: 案件化解后的状态，达成动态平衡
- `name`: 状态名称，简洁标识该阶段
- `status`: 状态描述，说明该阶段的特征（20-50 字）
- `leftWeights`: 驱动力列表，2-4 项，代表推动案件发生或升级的力量
- `rightWeights`: 约束力列表，2-5 项，代表抑制案件或促进化解的力量
- `value`: 权重数值，用于可视化展示，建议范围 10-60，同一状态内的权重值应合理分配
- `describe`: 权重项描述，说明该权重项的作用机制（30-80 字）
- `analysis`: 仅在"归因闭环"状态中包含，3-5 条分析要点，说明平衡达成的关键因素

**注意**: 
- 从"失衡诱发"到"归因闭环"，右侧约束力的总权重应逐渐增加，最终超过或接近左侧驱动力
- 权重值主要用于可视化展示，应保持合理的相对比例关系
